# Security Policy

## Reporting a Vulnerability

Please report vulnerabilities privately by emailing:

- support@kuzenbo.com

Do not open public issues for security reports.

## Scope

- `@kuzenbo/ai`
- `@kuzenbo/charts`
- `@kuzenbo/core`
- `@kuzenbo/code`
- `@kuzenbo/datatable`
- `@kuzenbo/date`
- `@kuzenbo/hooks`
- `@kuzenbo/notifications`
- `@kuzenbo/styles`
- `@kuzenbo/theme`

## What to Include

- Description of the issue
- Reproduction steps
- Potential impact
- Any mitigation ideas (if available)
- Affected package/version range (if known)

## Severity Expectations

- Critical: remote code execution, privilege escalation, credential compromise
- High: data exposure, integrity compromise, supply-chain execution risk
- Medium: significant misuse paths with realistic preconditions
- Low: edge-case weaknesses with limited impact

Severity is triaged by maintainers and may be adjusted during investigation.

## Response Targets

- Initial response: within 3 business days
- Ongoing updates: as investigation progresses

## Disclosure

We follow coordinated disclosure:

1. We confirm, triage, and reproduce the issue.
2. We prepare and ship a fix.
3. We publish remediation notes with affected versions and upgrade guidance.
