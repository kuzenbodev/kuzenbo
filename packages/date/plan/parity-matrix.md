# Parity Matrix (Mantine -> Kuzenbo)

| Mantine surface                                          | Kuzenbo surface                           | Parity requirement                                                                                                                                                       |
| -------------------------------------------------------- | ----------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `DatesProvider` + `useDatesContext`                      | `DatesProvider` + `useDatesContext`       | Same runtime override model for locale/week start/weekend/label separator/consistent weeks + timezone/direction extensions                                               |
| `Calendar`                                               | `Calendar`                                | Month/year/decade levels, controlled/uncontrolled view date+level, min/max level clamps, numberOfColumns/columnsToScroll, next/prev callbacks, keyboard shortcuts parity |
| `MonthLevelGroup/YearLevelGroup/DecadeLevelGroup`        | same names                                | Same multi-column rendering and cross-column arrow traversal                                                                                                             |
| `MonthLevel/YearLevel/DecadeLevel`                       | same names                                | Same header-label formatting callbacks and disabled nav logic against min/max bounds                                                                                     |
| `Month`, `Day`, `WeekdaysRow`                            | same names                                | Same grid generation, outside/hidden behavior, weekend detection, week numbers, aria-label defaults/overrides                                                            |
| `YearsList`, `MonthsList`, `PickerControl`               | same names                                | Same tab-stop heuristics (selected -> current -> first enabled), disabled calculations, keyboard movement                                                                |
| `DatePicker`                                             | `DatePicker`                              | Single/multiple/range behavior parity, hover preview, presets, controlled/uncontrolled parity                                                                            |
| `YearPicker`, `MonthPicker`                              | same names                                | Same selection semantics as `DatePicker` at year/month granularity                                                                                                       |
| `DatePickerInput`, `YearPickerInput`, `MonthPickerInput` | same names                                | Same dropdown behavior, clear behavior, format/value text behavior, close-on-change semantics                                                                            |
| `DateInput`                                              | `DateInput`                               | Free-typed parsing behavior, fix-on-blur, clearability/deselect behavior, dropdown interactions, hidden input parity                                                     |
| `DateTimePicker`                                         | `DateTimePicker`                          | Date + time composition parity, submit/enter close behaviors, min/max time-from-date bounds, hidden input parity                                                         |
| `TimeInput`                                              | `TimeInput`                               | Native time input wrapper behavior and min/max blur clamping parity                                                                                                      |
| `TimePicker` + `SpinInput`                               | `TimePicker` + internal spin primitive    | Same keyboard model, focus transitions, 12h/24h semantics, paste behavior, partial validity behavior                                                                     |
| `TimeGrid` + compare helpers                             | `TimeGrid`, `isTimeBefore`, `isTimeAfter` | Same selection and disable matrix parity                                                                                                                                 |
| `TimeValue`                                              | `TimeValue`                               | Same formatting of string/date inputs, 12h edge cases                                                                                                                    |
| `HiddenDatesInput`                                       | `HiddenDatesInput`                        | Exact formatting contract for single/multiple/range and with-time                                                                                                        |
| exported hooks/utils/types                               | same names                                | Maintain behavior contracts while implemented via adapter (no dayjs)                                                                                                     |
