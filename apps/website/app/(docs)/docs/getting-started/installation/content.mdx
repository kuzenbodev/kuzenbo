In almost all cases, keep `@kuzenbo/theme` installed with every Kuzenbo UI package you use.

## Add styles

Import theme CSS first, then optional baseline styles. In your main entry or global CSS file:

```tsx
import "@kuzenbo/theme/prebuilt/kuzenbo.css";
import "@kuzenbo/styles/recommended.css";
```

Theme CSS must come first so tokens load before the baseline overrides.

## Set up theme

For apps with SSR (e.g. Next.js App Router), add the bootstrap script and provider to your root layout:

```tsx
import type { ReactNode } from "react";

import { ThemeBootstrapScript, ThemeProvider } from "@kuzenbo/theme";

export default function RootLayout({ children }: { children: ReactNode }) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body>
        <ThemeBootstrapScript />
        <ThemeProvider>{children}</ThemeProvider>
      </body>
    </html>
  );
}
```

The script runs before React hydrates so dark mode applies on first paint without a flash.

## Verify your setup

Try rendering a button and card:

```tsx
import { Button } from "@kuzenbo/core/ui/button";
import { Card } from "@kuzenbo/core/ui/card";

export function InstallSmokeTest() {
  return (
    <Card>
      <Button>Install verified</Button>
    </Card>
  );
}
```

If you see them styled correctly, you're good to go.

## Troubleshooting

- **Components look unstyled** — `@kuzenbo/theme` must be installed with `@kuzenbo/core`. Import theme CSS before baseline styles.
- **Wrong theme on first load** — Ensure `ThemeBootstrapScript` runs before your app content (e.g. in `head` or top of `body`).
- **Legacy CSS paths** — Older paths like `@kuzenbo/core/styles.css` were removed. Use `@kuzenbo/theme/prebuilt/kuzenbo.css` and `@kuzenbo/styles/recommended.css`.
