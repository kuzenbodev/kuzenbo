## What this hook does

`useIsTooltipActive` is a Recharts v3 interaction hook re-exported by `@kuzenbo/charts`.
It reports whether tooltip interaction is currently active.

## Reads and writes

- Reads: tooltip interaction flag from Recharts chart state.
- Writes: nothing. This hook is read-only.

## When to use it

- Show hover-only helper UI when tooltip interaction is active.
- Skip expensive tooltip-dependent calculations when inactive.
- Coordinate custom tooltip content with chart cursor behavior.

## Import

```tsx
import { useIsTooltipActive } from "@kuzenbo/charts/ui/chart";
```

## Example: interaction-aware helper text

```tsx
"use client";

import { useIsTooltipActive } from "@kuzenbo/charts/ui/chart";

export function TooltipStateBadge() {
  const active = useIsTooltipActive();

  return (
    <span className="rounded-md border border-border px-2 py-1 text-xs">
      Tooltip: {active ? "active" : "idle"}
    </span>
  );
}
```

## Example: pair with active payload hook

```tsx
import {
  useActiveTooltipDataPoints,
  useIsTooltipActive,
} from "@kuzenbo/charts/ui/chart";

const active = useIsTooltipActive();
const points = useActiveTooltipDataPoints<{ value: number }>();
const visiblePoints = active ? (points ?? []) : [];
```

## Return contract and edge cases

- Type: `boolean`
- Returns `true` only while tooltip interaction is active.
- Returns `false` when interaction is inactive.
- Returns `false` when called outside chart context.
- Recharts expects one tooltip per chart; multiple tooltips can produce unpredictable state.

## SSR and RSC caveats

- Client-only runtime behavior.
- Keep interaction-driven rendering inside `"use client"` components.
- Use conservative fallbacks for initial render (`false`).
