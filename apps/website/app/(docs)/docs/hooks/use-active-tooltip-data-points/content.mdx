description: "Read the currently active tooltip payload objects from Recharts state during chart interaction."
---

## What this hook does

`useActiveTooltipDataPoints` is a direct Recharts v3 hook re-export from `@kuzenbo/charts`.
It returns the data records that are currently represented by the active tooltip.

## Reads and writes

- Reads: active tooltip payload from Recharts chart state.
- Writes: nothing. This hook is read-only.

## When to use it

- Build custom tooltip panels that need full data records, not only label text.
- Render side panels or badges that track the hovered point across multiple series.
- Derive cross-series metrics while tooltip `shared` mode is active.

## Import

```tsx
import { useActiveTooltipDataPoints } from "@kuzenbo/charts/ui/chart";
```

## Example: active payload summary

```tsx
"use client";

import {
  useActiveTooltipDataPoints,
  useIsTooltipActive,
} from "@kuzenbo/charts/ui/chart";

interface RevenuePoint {
  month: string;
  revenue: number;
  target: number;
}

export function ActivePayloadSummary() {
  const isTooltipActive = useIsTooltipActive();
  const points = useActiveTooltipDataPoints<RevenuePoint>();

  if (!isTooltipActive || points === undefined || points.length === 0) {
    return (
      <p className="text-sm text-muted-foreground">
        Hover the chart to inspect values.
      </p>
    );
  }

  return (
    <ul className="space-y-1 text-sm">
      {points.map((point) => (
        <li key={`${point.month}-${point.revenue}`}>
          {point.month}: revenue {point.revenue}, target {point.target}
        </li>
      ))}
    </ul>
  );
}
```

## Example: derive one active record safely

```tsx
const points = useActiveTooltipDataPoints<{ revenue: number }>();
const activeRevenue = points?.[0]?.revenue ?? null;
```

## Return contract and edge cases

- Type: `ReadonlyArray<T> | undefined`
- `undefined` when no active interaction is present.
- `undefined` when used outside a Recharts chart context.
- Array items come from tooltip payload records (`payload.map((entry) => entry.payload)`).
- Recharts deduplicates payload objects by reference before returning them.

## SSR and RSC caveats

- Use this hook in Client Components only.
- In Next.js App Router, keep chart interaction UI behind a `"use client"` boundary.
- On first render without interaction, expect `undefined` and branch explicitly.
