## Support Channels

Choose the channel that matches your issue:

- Bug: open a GitHub bug report.
- Feature request: open a GitHub feature request.
- Docs issue: open a documentation improvement request.
- Security issue: report privately to `support@kuzenbo.com`.

## Before You Open An Issue

1. Confirm the exact package and version.
2. Reproduce the issue in a minimal example.
3. Capture expected behavior and actual behavior.
4. Include environment details (OS, package manager, runtime/framework versions, browser).

## High-Quality Report Checklist

- Package name and installed version.
- Smallest reproducible snippet or repository.
- Clear step-by-step reproduction.
- Screenshots, logs, or stack traces when available.
- Impact summary (what is blocked in your app).

## Security Reporting

Do not post security vulnerabilities publicly.
Send details privately to `support@kuzenbo.com`, including:

- affected package and version,
- reproduction steps,
- impact assessment,
- any temporary mitigation you found.

See [SECURITY.md](https://github.com/kuzenbodev/kuzenbo/blob/main/SECURITY.md) for full policy details.
