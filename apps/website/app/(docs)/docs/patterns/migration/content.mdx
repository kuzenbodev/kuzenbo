## Migration Strategy

Migrate in small, verifiable steps.
Start with imports and runtime setup, then update feature areas one domain at a time.
In almost all apps, keep `@kuzenbo/theme` paired with Kuzenbo UI packages throughout the migration.

## Package Mapping

| Legacy usage                       | Recommended package                                                                  |
| ---------------------------------- | ------------------------------------------------------------------------------------ |
| chart APIs from `@kuzenbo/core`    | `@kuzenbo/charts`                                                                    |
| toast APIs from `@kuzenbo/core`    | `@kuzenbo/notifications`                                                             |
| calendar APIs from `@kuzenbo/core` | `@kuzenbo/date`                                                                      |
| legacy core CSS entrypoint         | `@kuzenbo/theme/prebuilt/kuzenbo.css` (+ optional `@kuzenbo/styles/recommended.css`) |

## Install Updated Packages

```bash
bun add @kuzenbo/core @kuzenbo/theme @kuzenbo/charts @kuzenbo/notifications @kuzenbo/date
```

If you split work into multiple migration PRs, include `@kuzenbo/theme` in each UI package batch.

## Import Migration Example

```tsx
// legacy (before, root import)
import { Calendar, LineChart, Toast } from "@kuzenbo/core";

// after (folder-mirrored subpaths)
import { LineChart } from "@kuzenbo/charts/ui/chart";
import { Toast } from "@kuzenbo/notifications/ui/toast";
import { Calendar } from "@kuzenbo/date/ui/calendar";
```

## Runtime and CSS Setup

```tsx
import "@kuzenbo/theme/prebuilt/kuzenbo.css";
import "@kuzenbo/styles/recommended.css";
```

```tsx
import { ThemeBootstrapScript, ThemeProvider } from "@kuzenbo/theme";
```

Render `ThemeBootstrapScript` before app content to avoid theme flicker on first paint.

## Migration Checklist

- Replace deep or legacy imports with package entrypoints.
- Update chart/date/notification usage to domain packages.
- Keep stories and tests aligned with API changes.
- Ship migrations in small pull requests for easier rollback.
