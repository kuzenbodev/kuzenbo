## Overview

`@kuzenbo/charts` provides ready-made chart components for fast implementation in product dashboards and reporting flows.

## Install

```bash
bun add @kuzenbo/charts @kuzenbo/theme recharts
```

In almost all apps, install `@kuzenbo/theme` together with `@kuzenbo/charts`.

## Example

```tsx
import { LineChart } from "@kuzenbo/charts/ui/line-chart";

const data = [
  { month: "Jan", revenue: 172_000, target: 180_000 },
  { month: "Feb", revenue: 186_000, target: 190_000 },
  { month: "Mar", revenue: 201_000, target: 205_000 },
];

export const RevenueVsTarget = () => (
  <LineChart
    chartRootProps={{ className: "h-72 w-full" }}
    data={data}
    dataKey="month"
    series={[
      { name: "revenue", label: "Revenue" },
      { name: "target", label: "Target", yAxisId: "right-axis" },
    ]}
    valueFormatter={(value) => `$${value.toLocaleString()}`}
    withLegend
    withRightYAxis
    rightYAxisProps={{ yAxisId: "right-axis" }}
  />
);
```

## Accessibility Notes

- Keep `accessibilityLayer` enabled for Recharts primitives.
- Use readable `series` labels so tooltips and legends communicate meaning.
- Do not rely on color alone to communicate important differences.

## Troubleshooting

- Missing colors usually means `@kuzenbo/theme` is not installed or theme CSS was not loaded.
- If a chart import fails, check that the subpath matches one of the public chart entrypoints.
