## Purpose

`Calendar` from `@kuzenbo/date` is a styled wrapper around `react-day-picker` with Kuzenbo tokenized styling and composable slot overrides.

It supports two distinct usage modes:

- Display-only month grid when `mode` is not set.
- Interactive picker when `mode` is set (`single`, `multiple`, `range`, etc. via DayPicker props).

## Installation

```npm
npm i @kuzenbo/date
```

## Import

```tsx
import {
  Calendar,
  CalendarChevron,
  CalendarDayButton,
  CalendarRoot,
  CalendarWeekNumber,
} from "@kuzenbo/date";
```

## Basic usage

Display-only calendar (no day buttons):

```tsx
import { Calendar } from "@kuzenbo/date";

export function BillingMonthPreview() {
  return (
    <Calendar className="rounded-lg border" defaultMonth={new Date(2025, 0)} />
  );
}
```

Single-date selection mode:

```tsx
import { useState } from "react";
import { Calendar } from "@kuzenbo/date";

export function SingleDatePicker() {
  const [selected, setSelected] = useState<Date | undefined>(new Date());

  return (
    <Calendar
      className="rounded-lg border"
      mode="single"
      onSelect={setSelected}
      selected={selected}
    />
  );
}
```

## Advanced usage: range, week numbers, locale, and dropdown caption

```tsx
import type { DateRange } from "react-day-picker";
import { useState } from "react";
import { arSA } from "react-day-picker/locale";

import { Calendar } from "@kuzenbo/date";

export function ReportingCalendar() {
  const [range, setRange] = useState<DateRange | undefined>({
    from: new Date(2025, 0, 8),
    to: new Date(2025, 0, 13),
  });

  return (
    <Calendar
      buttonVariant="ghost"
      captionLayout="dropdown"
      className="rounded-lg border"
      defaultMonth={range?.from}
      dir="rtl"
      locale={arSA}
      mode="range"
      numberOfMonths={2}
      onSelect={setRange}
      selected={range}
      showWeekNumber
    />
  );
}
```

## API reference

- `Calendar`
  Main export. Accepts `DayPicker` props plus `buttonVariant` (`ghost`, etc. from calendar button variants) and `size`.
- `CalendarRoot`
  Root slot override used internally; exported for advanced customization.
- `CalendarChevron`
  Navigation chevron component.
- `CalendarDayButton`
  Interactive day button renderer used when selection mode is enabled.
- `CalendarWeekNumber`
  Week-number cell renderer used with `showWeekNumber`. It follows the same `xs`â†’`xl` typography scale as the root calendar size.

## Size Scale

`Calendar` supports:

- `xs`
- `sm`
- `md` (default)
- `lg`
- `xl`

The `size` prop scales:

- root spacing
- cell size (`--cell-size`)
- day typography and controls
- navigation buttons and chevrons
- week number density

## Accessibility notes

- Provide surrounding label/context when used as a form field (for example, a visible `<label>` or field title).
- In interactive modes, keep `mode` explicit and ensure disabled dates are communicated by surrounding helper text.
- For locale-sensitive UIs, pass both `locale` and `dir` when RTL languages are expected.

## SSR and RSC notes

`Calendar` is a client component (`"use client"`).

In App Router, load initial date defaults on the server if needed, but render `Calendar` inside a client boundary for selection and navigation events.

## Styling and semantic tokens

The component is already wired to semantic tokens (`bg-background`, `text-foreground`, `bg-muted`, status tokens). Prefer token-based overrides through `className` or `classNames` slots.

If you customize day cells, keep both `day` and `day_button` slots coherent to avoid regressions between display-only and interactive modes.

## Troubleshooting

- Days are not clickable:
  Set a selection `mode` such as `single` or `range`.
- Outside days disappear unexpectedly:
  Check `showOutsideDays` (defaults to `true` here).
- Range visuals look wrong after custom styles:
  Preserve range classes (`range_start`, `range_middle`, `range_end`) when overriding `classNames`.
- Week numbers not visible:
  Enable `showWeekNumber`.
