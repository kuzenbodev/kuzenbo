## Components Overview

This section documents the component surfaces in Kuzenbo's publishable UI packages.
Use it to choose the right package entrypoint and compose features safely.

## Package Boundaries

- `@kuzenbo/core`: foundational UI primitives and composition utilities.
- `@kuzenbo/charts`: chart primitives and prebuilt chart components.
- `@kuzenbo/code`: developer-experience code surfaces and Shiki-powered docs primitives.
- `@kuzenbo/date`: calendar/day-picker surface.
- `@kuzenbo/notifications`: toast primitives and toast manager helpers.
- `@kuzenbo/ai`: AI widget, prompt builder, and AI session hooks.
- `@kuzenbo/datatable`: mock datatable primitive and datatable state utilities.
- `@kuzenbo/hooks`: standalone reusable hooks.
- `@kuzenbo/theme`: runtime theme provider/bootstrap + theme CSS.
- `@kuzenbo/styles`: optional baseline stylesheet (`recommended.css`).

## Install by Surface

In almost all apps, pair Kuzenbo UI packages with `@kuzenbo/theme`.

```npm
npm i @kuzenbo/core @kuzenbo/theme @kuzenbo/styles
```

```npm
npm i @kuzenbo/theme @kuzenbo/charts @kuzenbo/date @kuzenbo/notifications
```

```npm
npm i @kuzenbo/theme @kuzenbo/ai @kuzenbo/datatable @kuzenbo/hooks
```

```npm
npm i @kuzenbo/theme @kuzenbo/core @kuzenbo/code
```

If you only consume `@kuzenbo/hooks` without Kuzenbo UI packages, `@kuzenbo/theme` is usually optional.

## Basic Composition Example

```tsx
import { Button } from "@kuzenbo/core/ui/button";
import { Typography } from "@kuzenbo/core/ui/typography";
import { Calendar } from "@kuzenbo/date/ui/calendar";

export function PlannerPanel() {
  return (
    <section className="space-y-4 rounded-lg border border-border bg-card p-4">
      <Typography.H3>Release Planner</Typography.H3>
      <Calendar mode="single" />
      <Button size="sm">Create Release</Button>
    </section>
  );
}
```

## Advanced Composition Example

```tsx
"use client";

import "@kuzenbo/styles/recommended.css";
import "@kuzenbo/theme/default.css";

import { Button } from "@kuzenbo/core/ui/button";
import { Toast, useToast } from "@kuzenbo/notifications/ui/toast";
import { ThemeProvider } from "@kuzenbo/theme";

const SaveButton = () => {
  const toast = useToast();

  return (
    <Button
      onClick={() =>
        toast.success({
          title: "Saved",
          description: "Preferences synced successfully.",
        })
      }
    >
      Save Preferences
    </Button>
  );
};

export function AppShell() {
  return (
    <ThemeProvider>
      <Toast.Provider>
        <SaveButton />
        <Toast.Viewport />
      </Toast.Provider>
    </ThemeProvider>
  );
}
```

## API and Import Rules

- Import from package roots only (`@kuzenbo/core`, not `@kuzenbo/core/src/...`).
- Keep chart imports in `@kuzenbo/charts` and toast imports in `@kuzenbo/notifications`.
- Keep hooks imported from `@kuzenbo/hooks` instead of re-exporting them through UI packages.
- Keep theme runtime (`@kuzenbo/theme`) and baseline css (`@kuzenbo/styles`) concerns separate.

## Accessibility and UX

- Follow each component page for keyboard and screen-reader specifics.
- Prefer semantic typography/color primitives from `@kuzenbo/core` across packages.
- Treat loading and empty states as first-class UX paths, not afterthoughts.

## SSR and RSC Notes

- Server components can render static shells and pass serializable props.
- Interactive surfaces (menus, toasts, combobox, drag/drop interactions) must live in client components.
- Theme bootstrapping should happen before paint when SSR consistency is required.

## Styling and Tokens

- Use semantic tokens (`bg-background`, `text-foreground`, `border-border`, `text-muted-foreground`).
- Use package-level APIs before writing app-local utility overrides.
- Avoid raw palette classes to preserve theme parity across packages.

## Troubleshooting

- If imports fail, verify package ownership and avoid cross-package internal paths.
- If styling drifts, confirm theme css and optional baseline css are loaded in the expected order.
- If hydration mismatches appear, move interactive logic behind explicit client boundaries.
