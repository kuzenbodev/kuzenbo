## Dropdown Menu at a Glance

`DropdownMenu` is the canonical contextual action surface in `@kuzenbo/core`.
It supports checkbox/radio rows, submenu trees, link items, and slot-level composition primitives for custom layouts.

Use it for command lists and per-item action menus.
If you need persistent settings content, prefer `Dialog`, `Popover`, or `Sheet`.

## Installation

```npm
npm i @kuzenbo/core
```

## Import

```tsx
import { Button } from "@kuzenbo/core/ui/button";
```

## Basic Usage

```tsx
import { Button } from "@kuzenbo/core/ui/button";
import { DropdownMenu } from "@kuzenbo/core/ui/dropdown-menu";

export function ProjectActionsMenu() {
  return (
    <DropdownMenu>
      <DropdownMenu.Trigger render={<Button variant="outline" />}>
        Actions
      </DropdownMenu.Trigger>

      <DropdownMenu.Content align="end">
        <DropdownMenu.Item>Rename</DropdownMenu.Item>
        <DropdownMenu.Item>Duplicate</DropdownMenu.Item>
        <DropdownMenu.Separator />
        <DropdownMenu.Item variant="danger">Delete</DropdownMenu.Item>
      </DropdownMenu.Content>
    </DropdownMenu>
  );
}
```

## Advanced Usage

```tsx
"use client";

import { useState } from "react";

import { Button } from "@kuzenbo/core/ui/button";
import { DropdownMenu } from "@kuzenbo/core/ui/dropdown-menu";

export function ViewPreferencesMenu() {
  const [showHidden, setShowHidden] = useState(false);
  const [density, setDensity] = useState("comfortable");

  return (
    <DropdownMenu>
      <DropdownMenu.Trigger render={<Button variant="outline" />}>
        View
      </DropdownMenu.Trigger>

      <DropdownMenu.Content align="end" sideOffset={6}>
        <DropdownMenu.Label>Display</DropdownMenu.Label>

        <DropdownMenu.CheckboxItem
          checked={showHidden}
          onCheckedChange={setShowHidden}
        >
          Show hidden files
        </DropdownMenu.CheckboxItem>

        <DropdownMenu.Separator />

        <DropdownMenu.RadioGroup onValueChange={setDensity} value={density}>
          <DropdownMenu.RadioItem value="compact">
            Compact
          </DropdownMenu.RadioItem>
          <DropdownMenu.RadioItem value="comfortable">
            Comfortable
          </DropdownMenu.RadioItem>
        </DropdownMenu.RadioGroup>

        <DropdownMenu.Separator />

        <DropdownMenu.Sub>
          <DropdownMenu.SubTrigger>More Actions</DropdownMenu.SubTrigger>
          <DropdownMenu.SubContent>
            <DropdownMenu.Item variant="danger">
              Archive Project
            </DropdownMenu.Item>
          </DropdownMenu.SubContent>
        </DropdownMenu.Sub>
      </DropdownMenu.Content>
    </DropdownMenu>
  );
}
```

## Size Scaling

`DropdownMenu` supports `size="xs" | "sm" | "md" | "lg" | "xl"` with `md` as default.

- Root-driven default: `DropdownMenu size` sets the baseline for popup, label, item, shortcut, and submenu surfaces.
- Content override: `DropdownMenu.Content size` overrides root size for overlay descendants.
- Child override: item-level props (`DropdownMenu.Item size`, `DropdownMenu.Label size`, etc.) override both content and root.

Precedence:

- Overlay descendants: `explicit child size ?? nearest content size ?? root size ?? "md"`.

```tsx
<DropdownMenu size="sm">
  <DropdownMenu.Trigger>Actions</DropdownMenu.Trigger>
  <DropdownMenu.Content size="lg">
    <DropdownMenu.Item size="xl">Delete</DropdownMenu.Item>
  </DropdownMenu.Content>
</DropdownMenu>
```

## API Details

- Root: `DropdownMenu`.
- Trigger/content: `DropdownMenuTrigger`, `DropdownMenuContent`.
- Portal and positioning: `DropdownMenuPortal`, `DropdownMenuPopup`, `DropdownMenuPositioner`, `DropdownMenuArrow`, `DropdownMenuBackdrop`.
- Item primitives: `DropdownMenuItem`, `DropdownMenuCheckboxItem`, `DropdownMenuRadioItem`, `DropdownMenuLinkItem`.
- Grouping helpers: `DropdownMenuGroup`, `DropdownMenuLabel`, `DropdownMenuSeparator`, `DropdownMenuShortcut`.
- Nested menus: `DropdownMenuSub`, `DropdownMenuSubTrigger`, `DropdownMenuSubContent`.

## Accessibility

- Keep trigger labels explicit so screen readers can identify intent.
- Preserve visible focus states for keyboard users.
- Use clear wording for destructive actions.
- Keep menu items concise and action-oriented.

## SSR and RSC Notes

- Menu interactions are client-side.
- Server rendering is safe for shell markup, while open/close and selection behavior need a client boundary.

## Styling and Tokens

- Menu surfaces use semantic overlay tokens (`bg-popover`, `text-popover-foreground`, `ring-foreground/10`).
- Prefer semantic variants (`variant="danger"`) over hard-coded palette classes.
- Use slot-level overrides on content and item primitives for controlled customization.

## Troubleshooting

- If clipping occurs, check ancestor overflow styles and portal usage.
- If submenu alignment feels off, tune `alignOffset` and `sideOffset` on submenu content.
- If an item should stay open on select, handle `onSelect` and prevent default close behavior.
