# Button

`Button` is the main action primitive in `@kuzenbo/core`.
It supports semantic variants, a shared text-action size scale, and a built-in loading state.
For icon-only actions, use [`ActionIcon`](/docs/components/action-icon).

Use `Button` for commands that trigger behavior.
For plain navigation, prefer links unless the interaction is command-like (submit, save, delete, retry, confirm).

## Leading and trailing content

`Button` does not use `leftSection` / `rightSection` props.
Compose icons or other elements directly inside `children`:

<ButtonInlineContentComponentPreview />

```tsx
import { ArrowRight01Icon, SearchIcon } from "@hugeicons/core-free-icons";
import { HugeiconsIcon } from "@hugeicons/react";
import { Button } from "@kuzenbo/core/ui/button";

export function ButtonWithInlineContent() {
  return (
    <div className="flex flex-wrap gap-2">
      <Button variant="outline">
        <HugeiconsIcon icon={SearchIcon} />
        Find record
      </Button>

      <Button>
        Continue
        <HugeiconsIcon icon={ArrowRight01Icon} />
      </Button>
    </div>
  );
}
```

## Loading state

Set `isLoading` to replace content with a centered spinner and disable interaction.
`isLoading` also sets `disabled` under the hood, so click/submit handlers do not fire.

```tsx
import { Button } from "@kuzenbo/core/ui/button";

export function LoadingButton() {
  return <Button isLoading>Saving workspace</Button>;
}
```

## Disabled state

Use `disabled` for non-interactive buttons.

```tsx
import { Button } from "@kuzenbo/core/ui/button";

export function DisabledButton() {
  return <Button disabled>Access restricted</Button>;
}
```

## Render as link

`Button` forwards Base UI Button primitive props, including `render`.
When rendering as `<a>`, remember that links do not support native `disabled`.

```tsx
import { Button } from "@kuzenbo/core/ui/button";

type DocsLinkButtonProps = {
  isDisabled?: boolean;
};

export function DocsLinkButton({ isDisabled = false }: DocsLinkButtonProps) {
  return (
    <Button
      aria-disabled={isDisabled}
      className={isDisabled ? "pointer-events-none" : undefined}
      render={
        <a
          href="/docs/components/button-group"
          onClick={(event) => {
            if (isDisabled) {
              event.preventDefault();
            }
          }}
        />
      }
      variant="link"
    >
      Open ButtonGroup docs
    </Button>
  );
}
```

## ButtonGroup

Use `ButtonGroup` to merge adjacent button borders and radii for action clusters.
Direct child slots should be Kuzenbo controls (`Button`, `ButtonGroupText`, `ButtonGroupSeparator`) to preserve expected borders.

### Vertical groups and section slots

```tsx
import { Button } from "@kuzenbo/core/ui/button";
import {
  ButtonGroup,
  ButtonGroupSeparator,
  ButtonGroupText,
} from "@kuzenbo/core/ui/button-group";

export function VerticalGroup() {
  return (
    <ButtonGroup
      aria-label="Deployment actions"
      orientation="vertical"
      role="group"
      size="sm"
    >
      <Button variant="outline">Deploy preview</Button>
      <ButtonGroupSeparator orientation="horizontal" />
      <Button variant="outline">Run smoke tests</Button>
      <ButtonGroupText render={<div />}>Auto-sync enabled</ButtonGroupText>
    </ButtonGroup>
  );
}
```
