## Overview

`Chart` is the primitive composition layer in `@kuzenbo/charts`. Use it when prebuilt charts are too opinionated and you need direct Recharts control while keeping Kuzenbo tooltip, legend, and tokenized series color behavior.

You can compose primitives in two ways:

- `ChartRoot` path: fastest setup (`config` + frame + autosize included).
- manual path: `ChartProvider` + `ChartFrame` + `ChartAutoSize` for full control.

## Installation

```npm
npm i @kuzenbo/charts recharts
```

## Import

```tsx
import {
  Chart,
  ChartAutoSize,
  ChartFrame,
  ChartLegend,
  ChartLegendContent,
  ChartPortalTarget,
  ChartProvider,
  ChartRoot,
  ChartStyle,
  ChartTooltip,
  ChartTooltipContent,
  type ChartConfig,
} from "@kuzenbo/charts";
```

## Basic Example

```tsx
import {
  ChartRoot,
  ChartLegend,
  ChartLegendContent,
  ChartTooltip,
  ChartTooltipContent,
  type ChartConfig,
} from "@kuzenbo/charts";
import { Bar, BarChart, CartesianGrid, XAxis, YAxis } from "recharts";

const data = [
  { month: "Jan", visits: 120, revenue: 4500 },
  { month: "Feb", visits: 148, revenue: 5200 },
  { month: "Mar", visits: 170, revenue: 6100 },
];

const config: ChartConfig = {
  visits: { label: "Visits", color: "var(--color-chart-1)" },
  revenue: { label: "Revenue", color: "var(--color-chart-2)" },
};

export function PrimitiveChartBasic() {
  return (
    <ChartRoot className="h-80 w-full" config={config}>
      <BarChart data={data}>
        <CartesianGrid vertical={false} />
        <XAxis dataKey="month" />
        <YAxis />
        <Bar dataKey="visits" fill="var(--color-visits)" radius={6} />
        <Bar dataKey="revenue" fill="var(--color-revenue)" radius={6} />
        <ChartTooltip content={<ChartTooltipContent />} />
        <ChartLegend content={<ChartLegendContent />} />
      </BarChart>
    </ChartRoot>
  );
}
```

## Advanced Example

```tsx
import {
  ChartAutoSize,
  ChartFrame,
  ChartLegend,
  ChartLegendContent,
  ChartProvider,
  ChartTooltip,
  ChartTooltipContent,
  type ChartConfig,
} from "@kuzenbo/charts";
import { CartesianGrid, Line, LineChart, XAxis, YAxis } from "recharts";

const data = [
  { day: "Mon", signups: 18 },
  { day: "Tue", signups: 25 },
  { day: "Wed", signups: 22 },
  { day: "Thu", signups: 31 },
  { day: "Fri", signups: 28 },
];

const config: ChartConfig = {
  signups: {
    label: "Signups",
    theme: {
      light: "oklch(0.61 0.18 253)",
      dark: "oklch(0.74 0.16 253)",
    },
  },
};

export function PrimitiveChartAdvanced() {
  return (
    <ChartProvider config={config} id="signup-trend">
      <ChartFrame className="h-72 w-full">
        <ChartAutoSize>
          <LineChart data={data}>
            <CartesianGrid vertical={false} />
            <XAxis dataKey="day" />
            <YAxis />
            <Line dataKey="signups" dot={false} stroke="var(--color-signups)" />
            <ChartTooltip
              content={
                <ChartTooltipContent
                  unit="users"
                  valueFormatter={(value) => `${value.toFixed(0)}`}
                />
              }
            />
            <ChartLegend content={<ChartLegendContent />} />
          </LineChart>
        </ChartAutoSize>
      </ChartFrame>
    </ChartProvider>
  );
}
```

## API Reference

- `Chart`: namespace entry with attached primitives, hooks, and `Preset.*` shortcuts.
- `ChartRoot`: one-step wrapper (`ChartProvider` + `ChartFrame` + `ChartAutoSize`).
- `ChartProvider`: context provider for config, chart id, and series registry.
- `ChartFrame`: chart shell and Recharts styling contract layer.
- `ChartAutoSize`: optional `ResponsiveContainer` wrapper (`enabled` toggle).
- `ChartStyle`: emits per-chart CSS variables from config/theme mappings.
- `ChartTooltip`: re-exported Recharts `Tooltip`.
- `ChartTooltipContent`: Kuzenbo tooltip renderer with `unit`, `valueFormatter`, `indicator` controls.
- `ChartLegend`: re-exported Recharts `Legend`.
- `ChartLegendContent`: Kuzenbo legend renderer with payload normalization and optional highlight callback.
- `ChartPortalTarget`: target node for portal-based tooltip/overlay placement.

## Accessibility

- Keep clear `label` values in `ChartConfig` for tooltip/legend readability.
- Pair charts with nearby textual summary for screen reader context.
- Use tooltip/legend content that explains units (`%`, `USD`, `ms`) explicitly.
- Avoid encoding state by color alone; combine with label text and markers.

## SSR and RSC Notes

- Chart primitives are client-side (`"use client"`) due Recharts runtime behavior.
- Render them inside Client Components in App Router.
- Keep chart ids/config stable between renders to avoid style var churn.

## Styling and Tokens

- Provide series colors through `ChartConfig` using semantic chart vars (`var(--color-chart-1)`, etc).
- `ChartFrame` already maps axis/grid/tooltip visuals to semantic tokens.
- Keep page wrappers on semantic classes (`bg-background`, `text-foreground`, `border-border`).

## Troubleshooting

- `var(--color-*)` values not applied: verify config series keys match your Recharts `dataKey`.
- Legend labels fallback to raw keys: add `label` in `ChartConfig` entries.
- Tooltip row hidden unexpectedly: inspect payload type/color, rows with `none` are filtered.
- Chart clipping in small containers: set explicit height and check `ChartAutoSize` usage.
