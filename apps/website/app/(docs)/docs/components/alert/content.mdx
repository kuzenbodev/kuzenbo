## Overview

`Alert` and `AlertDialog` solve two different communication jobs:

- `Alert` is for inline, persistent feedback inside normal page flow.
- `AlertDialog` is for high-risk confirmations that should block progress until the user decides.

Use inline `Alert` for status, context, and guidance. Use `AlertDialog` only when an action has serious consequences.

## Installation

```npm
npm i @kuzenbo/core
```


## Basic Example

```tsx
import { Alert, AlertDescription, AlertTitle } from "@kuzenbo/core/ui/alert";

export function AccountAlert() {
  return (
    <Alert variant="warning">
      <AlertTitle>Plan limit reached</AlertTitle>
      <AlertDescription>
        Add seats or remove inactive users before inviting teammates.
      </AlertDescription>
    </Alert>
  );
}
```

## Advanced Example

```tsx
import { Button } from "@kuzenbo/core/ui/button";
import {
  AlertDialog,
  AlertDialogAction,
  AlertDialogCancel,
  AlertDialogContent,
  AlertDialogDescription,
  AlertDialogFooter,
  AlertDialogHeader,
  AlertDialogTitle,
  AlertDialogTrigger,
} from "@kuzenbo/core/ui/alert-dialog";

export function DeleteProjectDialog() {
  return (
    <AlertDialog>
      <AlertDialogTrigger render={<Button variant="danger" />}>
        Delete project
      </AlertDialogTrigger>

      <AlertDialogContent size="md">
        <AlertDialogHeader>
          <AlertDialogTitle>Delete this project?</AlertDialogTitle>
          <AlertDialogDescription>
            This permanently removes environments, API keys, and deployment
            logs.
          </AlertDialogDescription>
        </AlertDialogHeader>

        <AlertDialogFooter>
          <AlertDialogCancel>Cancel</AlertDialogCancel>
          <AlertDialogAction variant="danger">Delete</AlertDialogAction>
        </AlertDialogFooter>
      </AlertDialogContent>
    </AlertDialog>
  );
}
```

## API Reference

- `Alert`: inline container with `role="alert"`; supports `variant="default" | "primary" | "secondary" | "warning" | "danger" | "info" | "success"`, `appearance="default" | "subtle" | "outline" | "inverted"`, and `size="xs" | "sm" | "md" | "lg" | "xl"` (`md` default).
- `AlertTitle`: short, high-emphasis heading inside alert content.
- `AlertDescription`: longer supportive copy; link styles are included by default.
- `AlertAction`: right-aligned action slot for compact controls.
- `alertVariants`: variant helper when you need the same visual contract in custom wrappers.
- `AlertDialog`: confirm-flow root.
- `AlertDialogTrigger`: opener; supports Base UI `render` composition.
- `AlertDialogContent`: popup surface; supports `size="md" | "sm"`.
- `AlertDialogHeader`, `AlertDialogTitle`, `AlertDialogDescription`, `AlertDialogFooter`: semantic structure for confirm copy and controls.
- `AlertDialogCancel`, `AlertDialogAction`: prewired action buttons.
- `AlertDialogOverlay`, `AlertDialogPortal`: overlay and portal hooks.
- `AlertDialogMedia`: optional icon/media slot in the header area.

## Accessibility

- Inline alerts are announced via `role="alert"`; keep the message concise and actionable.
- Confirm flows render as `alertdialog`; always provide a title and description.
- Provide distinct button labels (`Cancel`, `Delete`, `Archive`) so intent is obvious to screen-reader users.

## SSR and RSC Notes

- `Alert` can be rendered in server components when content is static.
- `AlertDialog` is interactive and should be used from client components.
- In Next.js App Router, keep dialog state and callbacks behind a `"use client"` boundary.

## Styling and Tokens

- Use `variant` to choose semantic tone (`default`, `primary`, `secondary`, `warning`, `danger`, `info`, `success`).
- Use `appearance` to control emphasis (`default`, `subtle`, `outline`, `inverted`) without changing tone semantics.
- Use `size` to scale spacing, icon density, and text (`xs` through `xl`).
- `appearance="inverted"` gives a dark surface while keeping semantic tone accents through icon color.
- `Alert` styles use semantic tokens (`bg-card`, `bg-warning`, `bg-danger`, `text-foreground`, `text-muted-foreground`, `bg-background`).
- If you need custom emphasis, keep semantic token classes such as `border-border`, `text-muted-foreground`, and `bg-background`.

## Troubleshooting

- Alert looks muted in dark mode: confirm you are using `variant` and `appearance` instead of hard-coded palette classes.
- Dialog does not open: verify `AlertDialogTrigger` is inside `AlertDialog` and rendered as an interactive element.
- Cancel/action buttons are unstyled: use `AlertDialogCancel` and `AlertDialogAction` instead of raw buttons.
